HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The doctype declaration lets the computer know what document is being used, in this case HTML
  • lang= "en" lets the system know what language is being used for the website
  • The title element defines the title for the webpage- extremely important for Search Engine Optimization (SEO)
  • Examples of characteristic elements are as follows: "style", "base","link", "meta","script","viewpoint", etc.
  • Uses descriptors to form website information, such as, height and width of JPEGS, languages used, headers, footers, paragraphs of text, etc.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • declarations contain two important sections:the CSS property we want to apply and the value of that property. Ex: color: blue;
  • Margin property- acts like margin in word document. not visible on the page. Ex: if we set the margin to 0; there is no padding around the element
  • Padding property- adds space around the content inside of the element - increases size
  • Margin and padding properties are part of the declaration block for the * selector.
  • When combining two selectors into a style CSS rule with a comma, allowing the declarations to affect both selectors. Rather than repeating a rule which would add more laws of code and repetition, two or more elements that will share the same styling can be grouped together.
  • Do Not Repeat Yourself or DRY code - coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • img property assigns values to image element in the markup
  • diplay property- assinged with the block value , which assigns each image to start a new line.

Git

  • git add -A after any changes
  • git commit -m , ie. "fixed bug in html header"
  • git pull origin main - fetches commits from main, or master, branch of the origin remote
  • git push origin feature/starter-code - updates changes to existing gitHub repository
